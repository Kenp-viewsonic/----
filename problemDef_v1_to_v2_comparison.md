## 二、核心组件改造

### 2.1 KV前缀

| 维度 | v1 | v2 |
|---|---|---|
| 名称 | 可训练 KV 前缀 | **毒性语义锚定前缀（Toxic Semantic Anchor Prefix）** |
| 初始化 | 随机初始化 $\Theta_P^{(k)}$ | 用基类毒性样本 [CLS] 表示的 K-means 质心初始化 $P_{proto}$ |
| 更新公式 | $P^{(k)} = \Theta_P^{(k)}$ | $P^{(k)} = \alpha \cdot P_{proto} + (1-\alpha) \cdot \Theta_P^{(k)}$ |
| 场景论证 | 无 | 前缀不再是无先验的可训练参数，而是携带毒性语义结构信息的记忆锚点 |
| 增强机制 | "若后续需要样本自适应..."（模糊） | **演化感知更新**：当批次 OOV/字符变异密度高时，临时提升 $(1-\alpha)$ 以增强对表面形式变化的敏感度 |

### 2.2 LoRA适配

| 维度 | v1 | v2 |
|---|---|---|
| 名称 | QKV/FFN 固定秩 LoRA | **语义稳定性感知的双分支 LoRA（Semantic-Stability-Aware Dual-Branch LoRA）** |
| 结构 | 每阶段一组 LoRA（单一分支） | **两个低秩分支**：stable（Main）+ plastic（Dev） |
| stable分支 | 不存在 | 新增：学习跨阶段稳定的毒性语义核，强能量惩罚正则保护旧知识 |
| plastic分支 | 不存在 | 新增：学习当前阶段特有表面形式，L1稀疏约束防止过拟合 |
| 分支融合 | 简单累加 $\sum_{j\le k} \Delta W^{(j)}$ | $W_{eff}^{(k)} = W_{init} + \mathcal{W}_{past}^{stable} + \sum_{j \in \mathcal{F}_k} \Delta W_{plastic}^{(j)} + \Delta W_{plastic}^{(k)}$ |
| 核心机制 | 阶段冻结 | **语义沉淀（Semantic Consolidation）**：阶段结束时审查 plastic 分支对旧类的干扰 $\delta^{(k)}$，低干扰则合并到 stable，高干扰则隔离冻结 |
| 类别门控 | $\beta_c$（identity_hate 类更低，需人工预设） | **删除**。改为统一沉淀阈值 $\tau$，所有类别一视同仁，无需先验假设 |
| 推理参数 | 当前阶段 LoRA + 冻结历史 LoRA | 已沉淀 stable 核心 + 冻结历史 plastic 补丁 + 当前活跃 plastic 补丁 |
| 场景论证 | "低秩结构天然限制更新自由度"（通用） | 类比 git 分支工作流：dev 试验 → 审查 → 合并到 main / 保留为历史标签 |

### 2.3 评论结构感知位置编码

| 维度 | v1 | v2 |
|---|---|---|
| 名称 | 评论结构感知位置编码 | **毒性表达结构感知位置编码** |
| 信号 | $q_i$（引号/标点强度）、$m_i$（大写/强调）、$l_i$（句法片段） | 保留 $q_i, m_i, l_i$，**明确定义计算方式与毒性关联**（如重复标点与情绪爆发相关、反问句/条件威胁句的句法模板检测） |
| 新增信号 | 无 | **$v_i$（字符级变异标记）**：leet speak 替换（a→4, e→3）、空格规避、谐音替换等对抗性表面形式 |

### 2.4 开放集门控

| 维度 | v1 | v2 |
|---|---|---|
| 名称 | 开放集门控 | **变体感知层级拒识门控（Variant-Aware Hierarchical Rejection Gate）** |
| 未知概率输入 | max-sigmoid 置信度 + 多标签熵 + 原型距离 | **新增表面形式异常分数** $s_{surface}(x_t) = \text{CharEntropy} \cdot \text{OOV-Ratio} \cdot \max \text{EditSim}$ |
| 输出 | 单一未知概率 $u_t$ | **两级输出**：粗粒度（toxic vs non-toxic）+ 细粒度（"已知毒性框架的未知变体"细分提示） |
| 场景论证 | 通用 OOD 检测 | 针对毒性变体特点：已知毒性词的轻微字符扰动在嵌入空间中与已知类高度接近，仅靠语义距离不足，需引入表面形式异常通道 |

### 2.5 训练目标

| 维度 | v1 | v2 |
|---|---|---|
| 损失项 | 7项：BCE + ASL + Open + Distill + Orth + Imb + Prefix | **精简为5项**：BCE + $\mathcal{L}_{evo}$ + $\mathcal{L}_{stable/plastic}$ + Open + Orth |
| 核心新增 | 无 | **$\mathcal{L}_{evo}$（语义演化一致性损失）**：同一毒性语义的不同字符变体在 [CLS] 空间中靠近 |
| 删除项 | 无 | ASL（并入演化损失）、Imb（改用采样策略）、Distill（stable分支天然承担）、Prefix（并入演化损失） |
| 场景论证 | "分别对应..."（逐项罗列） | 明确优先级：3项核心（BCE+演化+稳定/可塑约束）+ 2项辅助（拒识+子空间去相关） |

## 四、删除/弱化内容

| 删除/弱化项 | v1 | v2 | 理由 |
|---|---|---|---|
| 类别级门控 $\beta_c$ | 按类别预设演化速度（identity_hate 更低） | **删除** | 需要人工先验假设，改为统一的语义沉淀机制 |
| ASL 损失 | 独立损失项 $\mathcal{L}_{asl}$ | **删除**（并入演化损失） | 演化一致性损失本身具有难样本聚焦效应，ASL 冗余 |
| 长尾损失 $\mathcal{L}_{imb}$ | 独立损失项 | **删除**（改用采样策略） | 工程层面用重采样替代损失加权，减少超参数量 |
| 蒸馏损失 $\mathcal{L}_{distill}$ | 独立损失项 | **删除**（stable分支天然承担） | stable分支的强子空间去相关已实现类似蒸馏的知识保护 |
| Prefix一致性损失 $\mathcal{L}_{prefix}$ | 独立损失项 | **删除**（并入演化损失） | $\mathcal{L}_{evo}$ 的表示约束已覆盖前缀条件一致性 |
| 动态 rank 分配 | "不引入动态 rank 分配与任务向量 SVD 机制" | **保留此声明** | 双分支 LoRA 的秩 $r_s, r_p$ 为固定超参，复杂度可控 |

## 五、一句话摘要对比

| 版本 | 摘要 |
|---|---|
| v1 | 本研究提出一个面向 FSCIL 的毒性评论识别框架，以"可训练 KV 前缀 + QKV/FFN 固定秩 LoRA 阶段冻结"为核心，通过评论结构感知位置编码与开放集门控实现"识别-分类-拒识-增量更新"闭环... |

- 新增"毒性表达持续演化""新黑话/变体辱骂"等场景关键词；
- 用"语义沉淀"替代了模糊的"阶段冻结"，明确知识管理机制。
